DUC 2002

Test Data

The test documents consist of 59 document sets, each in two formats - one with just the original TREC tagging, and one with rough sentence tagging added (for use in creating extracts). (Document set d088d had to be withdrawn because of problems with the derived manual abstracts - hence 59 docsets rather than the planned 60.) :

  • duc2002testdocs.tar.gz * - a gzipped tar file of the test documents
  • duc2002testdocswithsentences.tar.gz * - a gzipped tar file of the test documents with sentences tagged
The remaining test data directory/file structure is laid out as follows:

  • docset.categories : the category assignment for each document set. This was provided to the human summarizers along with the documents.
  • dtds/ : SGML DTDs for the various kinds of test documents in their basic TREC format (not with sentence tagging)
  • summaries/ :
    • summaries/ : manually created summaries
      • dnnnx:   document set number nnn (061-120) selected by assessor x
        (Note that there are duplicate extracts for all docsets except d076 and d098. In the latter two cases the missing extracts were withdrawn because, through an error, the set of document the assessor used was not identical to that provided to the systems. All extracts and abstracts for d088 were withdraawn for the same reason.)
        • 200e, 400e: the 200- and 400-word extracts for current document set
        • 10, 50, 100, 200: the 10-, 50-, 100, and 200-word abstracts for current document set
        • perdocs: single-document summaries for each document in current document set
    • extracts.dtd - SGML DTD for extracts
    • abstracts.dtd - SGML DTD for extracts
    • duc2002extracts.tar.gz - gzipped tar file of the extracts
    • duc2002abstracts.tar.gz - gzipped tar file of the abstracts
    • duc2002extractsandabstracts.tar.gz - gzipped tar file of the extracts and abstracts
